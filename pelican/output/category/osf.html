Introduction to the OSF

Thu 05 September 2013
By Melissa

In OSF.

tags: OSFhowto

This post is to be the first in a series on the Open Science Framework, intended in part to introduce new members to the endeavor and in part to describe a specific feature.

The OSF is infrastructure for scientists to organize and share their research. A researcher can use components (akin to directories) on the site for organizing projects at several levels of complexity: individual experiments, research areas, dissertations, and more. The file upload system allows researchers to share data (e.g. .csv), materials (e.g. JPGs of visual stimuli presented in a perception experiment), data analysis scripts, PDFs, and whatever else would facilitate collaboration. Projects can either be private and shared with specific collaborators registered on the site, or they can be made publicly available.

Project nesting

Here is an example of a simple project in the OSF. It’s for a replication, but it corresponds to one publication comprising one experiment. With a project as simple as this, it’s easy to create a dataset component and put in that component my single .csv file, and/or a materials component containing every media file used in the experiment, and so on.

As I continue research, though, I may want to differentiate my original and replication research. I can add projects to my main page and, inside those, add new components under the project category. As long as I make the category of a new component a project, I can continue adding components to that. Any other component type is necessarily terminal: when I make a materials component, my only option is to edit its wiki and to add files.

Contributors to the OSF are to set the standards for its use, and new possibilities arise with every lab! Let us know in the comments or by email if you have or know of an interesting use of the nesting feature of projects on the OSF.
